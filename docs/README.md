# Documentation

The credit-risk modeling work, written up one topic per file and ordered to follow the modeling
pipeline — from raw inputs through to governance. Each doc is self-contained, grounded in the
committed result tables (`modeling/model-results/`), and embeds its own chart. Read them in order for
the full story, or jump to the topic you need.

| # | Doc | What it covers |
|---|---|---|
| 01 | [Feature Engineering & Macro Overlay](01-feature-engineering.md) | IV/collinearity feature selection, the engineered affordability/credit ratios and `RiskCluster`, the v1→v4 ablation, and the point-in-time macro overlay — including how it's *TTC-anchored* to stay leakage-safe |
| 02 | [PD Model Finetuning](02-pd-model-finetuning.md) | the AutoML baseline, the XGBoost/LightGBM/RF challengers, isotonic calibration, and why the ~0.75 AUC ceiling is information-bound |
| 03 | [Validation Methodology](03-validation-methodology.md) | out-of-time (vintage) splits vs random, discrimination vs calibration, and the decile + by-vintage calibration checks |
| 04 | [Discrete-Time Hazard Model](04-discrete-time-hazard-model.md) | the loan-month panel, monthly hazard `h(t\|x)`, the PD term structure, and the scikit-survival benchmark |
| 05 | [ECL Backtesting](05-ecl-backtesting.md) | predicted `PD × LGD × EAD` vs realized dollar losses by vintage — the engine calibrated to within ~2% in dollars |
| 06 | [Model Card (SR 11-7)](06-model-card.md) | the governance summary: intended use, data, metrics, limitations, fair-lending posture, and why an SR 11-7 card matters |

## Charts

The committed figures, each embedded in the doc that explains it:

- `finetuning_matrix.png` — v1→v4 out-of-time challenger matrix (docs 01, 02)
